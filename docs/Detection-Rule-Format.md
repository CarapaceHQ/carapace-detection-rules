# Detection Rule Format

## Overview

Carapace uses structured detection rules to identify suspicious or malicious AI agent behavior.

Detection rules evaluate behavioral telemetry signals generated by the Carapace telemetry pipeline.

Rules may operate on:

* individual events
* aggregated behavioral windows
* correlated signals across multiple entities

Rules produce **detections, risk flags, and scoring inputs** that feed the Risk Scoring Engine.

The rule format is designed to be:

* human readable
* machine executable
* versioned
* shareable across organizations

Rule packs may be distributed through repositories such as:

* carapace-detection-rules
* carapace-community-rules

---

# Rule Structure

Each rule is defined as a structured object containing metadata, detection logic, and scoring impact.

Required rule fields include:

rule_id
title
description
category
severity
confidence
required_signals
logic
evaluation_window
scoring_impact
tags
version
author
enabled

---

# Rule Field Definitions

## rule_id

Unique identifier for the rule.

Example:

prompt_injection_basic

---

## title

Short human-readable rule name.

Example:

Prompt Injection Detection

---

## description

Detailed explanation of the rule and the behavior it detects.

---

## category

Detection category.

Examples include:

* network_abuse
* authentication_abuse
* request_behavior
* content_attack
* economic_abuse
* coordinated_activity

---

## severity

Indicates the potential seriousness of the behavior.

Possible values:

low
medium
high
critical

---

## confidence

Confidence score indicating how reliable the detection is.

Range:

0.0 to 1.0

Higher values indicate stronger confidence.

---

## required_signals

List of telemetry signals required for rule evaluation.

Example:

prompt_injection_signature
prompt_length_anomaly
request_velocity

---

## logic

Logical conditions that must be satisfied for the rule to trigger.

Logic may evaluate:

* event attributes
* aggregated metrics
* correlation signals

---

## evaluation_window

Defines the time window used for rule evaluation.

Examples:

event
1m
5m
15m
1h
24h

---

## scoring_impact

Defines how the detection influences the risk scoring engine.

Example:

content_risk +20

---

## tags

Optional tags used for grouping rules.

Examples:

prompt_security
api_protection
fraud_detection

---

## version

Version identifier for the rule.

Example:

v1.0

---

## author

Author or organization responsible for the rule.

Example:

carapace-core

---

## enabled

Boolean flag determining whether the rule is active.

---

# Example Rule — Prompt Injection Detection

rule_id: prompt_injection_basic

title: Prompt Injection Detection

description:
Detects known prompt injection patterns attempting to override model instructions or extract sensitive data.

category: content_attack

severity: high

confidence: 0.85

required_signals:

prompt_injection_signature
prompt_length_anomaly

logic:

prompt_injection_signature == true

evaluation_window: event

scoring_impact:

content_risk +20

tags:

prompt_security
llm_protection

version: v1.0

author: carapace-core

enabled: true

# Rule Packs

Detection rules may be distributed in rule packs.

Examples:

Core Rules

Maintained by the Carapace project.

Includes common abuse detections.

Community Rules

Maintained by security researchers and organizations.

Includes specialized detections and emerging threat indicators.

Organization Rules

Custom rules defined by individual organizations for internal policies.

---

# Rule Lifecycle

Detection rules follow a lifecycle.

Draft

Rule proposed and tested.

Review

Rule evaluated for accuracy and false positive rates.

Release

Rule included in official rule packs.

Monitoring

Rule performance monitored in production environments.

Revision

Rule updated as new attack patterns emerge.

---

# Rule Sharing Model

Organizations may share detection rules without exposing raw telemetry.

Shared rule packs enable collaborative security improvements across the ecosystem.

Rule sharing may include:

* behavioral detection heuristics
* prompt attack signatures
* infrastructure abuse indicators
* fraud detection patterns

This allows the Carapace ecosystem to evolve through shared intelligence.

---

# Summary

Detection rules provide the logic used by the Carapace Detection Engine to identify suspicious AI agent behavior.

The rule format enables:

* standardized detection logic
* rule sharing across organizations
* explainable detections
* versioned intelligence updates

Over time, rule packs become a key component of the Carapace security ecosystem.
